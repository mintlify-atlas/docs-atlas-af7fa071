## Overview

The Alacritty configuration includes an extensive collection of **151 professionally-crafted themes**, making it easy to customize your terminal's appearance to match your preferences or workflow.

## Theme Organization

Themes are organized in a dedicated directory structure:

```
~/.config/alacritty/themes/
├── themes/                     # Main themes directory
│   ├── coolnight.toml          # Current active theme
│   ├── tokyo_night.toml
│   ├── gruvbox_dark.toml
│   ├── solarized_dark.toml
│   ├── nord.toml
│   └── ... (151 total themes)
└── print_colors.sh             # Color preview utility
```

## How to Switch Themes

Switching themes in Alacritty is straightforward and takes effect immediately without restarting the terminal.

<Steps>
  <Step title="Browse Available Themes">
    List all available themes in the themes directory:
    ```bash
    ls ~/.config/alacritty/themes/themes/
    ```
    
    You'll see 151 theme files including:
    - Tokyo Night variants (storm, enhanced, default)
    - Gruvbox variants (dark, light, material)
    - Solarized (dark, light)
    - Nord and Nordic
    - GitHub themes (dark, light, high contrast)
    - Catppuccin variants
    - Dracula
    - One Dark / One Light
    - Monokai variants
    - And many more!
  </Step>

  <Step title="Edit Main Configuration">
    Open the main Alacritty configuration file:
    ```bash
    vim ~/.config/alacritty/alacritty.toml
    # or
    nvim ~/.config/alacritty/alacritty.toml
    ```
  </Step>

  <Step title="Update Theme Import">
    Modify the `general.import` line at the top of the file:
    ```toml
    general.import = [
        "~/.config/alacritty/themes/themes/YOUR_THEME.toml"
    ]
    ```
    
    For example, to switch to Tokyo Night:
    ```toml
    general.import = [
        "~/.config/alacritty/themes/themes/tokyo_night.toml"
    ]
    ```
  </Step>

  <Step title="Save and Enjoy">
    Save the file - Alacritty will automatically reload the configuration and apply the new theme immediately!
  </Step>
</Steps>

## Theme Preview Utility

The themes directory includes a `print_colors.sh` script for previewing color palettes:

```bash
# Preview colors in your current theme
bash ~/.config/alacritty/themes/print_colors.sh
```

This script displays all 16 terminal colors (8 normal + 8 bright) so you can see how your theme looks before committing to it.

## Creating Custom Themes

You can create your own custom theme by copying an existing one:

<Steps>
  <Step title="Copy a Base Theme">
    ```bash
    cp ~/.config/alacritty/themes/themes/coolnight.toml \
       ~/.config/alacritty/themes/themes/mytheme.toml
    ```
  </Step>

  <Step title="Edit Color Values">
    Modify the color values in your new theme file:
    ```toml
    [colors.primary]
    background = '#YOUR_BG_COLOR'
    foreground = '#YOUR_FG_COLOR'
    
    [colors.cursor]
    text = "#YOUR_CURSOR_TEXT"
    cursor = "#YOUR_CURSOR_COLOR"
    
    [colors.normal]
    black = "#YOUR_BLACK"
    red = "#YOUR_RED"
    green = "#YOUR_GREEN"
    # ... and so on
    ```
  </Step>

  <Step title="Apply Your Theme">
    Update your main configuration:
    ```toml
    general.import = [
        "~/.config/alacritty/themes/themes/mytheme.toml"
    ]
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Theme not loading">
    Verify the theme file path is correct:
    ```bash
    ls ~/.config/alacritty/themes/themes/YOUR_THEME.toml
    ```
    Check for typos in the theme filename in your `alacritty.toml`.
  </Accordion>

  <Accordion title="Colors look wrong">
    Ensure your `TERM` environment variable is set correctly:
    ```toml
    [ env ]
    TERM = "xterm-256color"
    ```
    This should be in your main `alacritty.toml` file.
  </Accordion>

  <Accordion title="Configuration not reloading">
    Alacritty auto-reloads, but if needed, manually reload with:
    - macOS: `Cmd + Q` then reopen
    - Linux: `Ctrl + Shift + Q` then reopen
    
    Or check for syntax errors in your TOML file.
  </Accordion>
</AccordionGroup>
